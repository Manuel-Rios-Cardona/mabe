Un sistema a la medida en Guatemala puede comenzar alrededor de **Q18,000 a Q45,000 para una primera versión operativa**. Una plataforma con varios módulos, migración e integraciones puede requerir **Q45,000 a Q120,000 o más**. Estos rangos son referencias para planificar, no una tarifa universal.

## Qué determina el costo

La cantidad de pantallas no es una medida suficiente. El presupuesto cambia según:

| Factor | Impacto habitual |
| --- | --- |
| Usuarios y roles | Aumenta permisos, pruebas y administración |
| Reglas y aprobaciones | Exige más escenarios y manejo de excepciones |
| Datos existentes | Puede requerir limpieza, mapeo y migración |
| Integraciones | Depende de APIs, proveedores y manejo de fallos |
| Seguridad y auditoría | Agrega controles, registros y pruebas |
| Soporte | Define monitoreo, respuesta y evolución |

Una empresa que necesita registrar órdenes con dos roles no tiene el mismo proyecto que una operación con sucursales, inventario, facturación y aprobaciones por monto.

## Rangos por etapa

### Diagnóstico: Q2,500 a Q6,000

Incluye entrevistas, mapa del proceso, usuarios, riesgos, datos, primera fase y criterios de aceptación. Esta etapa reduce cotizaciones imprecisas y permite comparar propuestas con el mismo alcance.

### Primera versión: Q18,000 a Q45,000

Suele cubrir un proceso prioritario, roles, formularios, estados, reportes básicos y despliegue. El objetivo es comprobar valor con usuarios reales antes de ampliar.

### Plataforma integrada: Q45,000 a Q120,000+

Puede incluir varios módulos, migración, APIs, trazabilidad, respaldos, monitoreo y operación crítica. Conviene dividirla en entregas verificables.

## Qué debe incluir una propuesta

Una cotización comparable identifica:

- Funciones incluidas y excluidas.
- Datos que se migrarán.
- Integraciones y costos de terceros.
- Ambientes, despliegue y respaldos.
- Pruebas y criterios de aceptación.
- Capacitación, garantía y soporte.
- Propiedad del código y documentación.

Evita comparar únicamente el precio final. Una propuesta económica puede omitir migración, seguridad o soporte y trasladar el costo al momento más difícil del proyecto.

## Cuánto tiempo requiere

Una primera fase suele tomar entre **6 y 14 semanas** después de validar alcance y datos. El calendario depende también de la disponibilidad de usuarios responsables para responder preguntas y aceptar entregas.

## Sistema propio, ERP o software existente

Un sistema propio conviene cuando el proceso diferencia al negocio o las herramientas disponibles obligan a mantener demasiado trabajo paralelo. Un ERP o producto estándar puede ser mejor cuando el proceso es común y la empresa acepta adaptarse.

Antes de decidir, compara licencias, implementación, integraciones, mantenimiento, dependencia del proveedor y costo del trabajo manual. Consulta la guía [sistema a la medida vs. ERP](/recursos/sistema-a-la-medida-vs-erp/) y la página de [desarrollo de sistemas a la medida](/servicios/sistemas-a-la-medida/).

## Cómo preparar una estimación confiable

Documenta durante dos semanas:

1. Quién ejecuta cada tarea.
2. Cuánto tiempo consume.
3. Qué errores y retrabajos aparecen.
4. Qué herramientas reciben o entregan datos.
5. Qué resultado indicará que el proyecto funcionó.
